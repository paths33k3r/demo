# DEMO build — AntiGravity Monthly Harvesting Report

A **self-contained, fictional-data** copy of the app for showing people how it
works **without exposing real plantation data**.

## What's different from the real app

| Aspect | Real app | This DEMO |
|---|---|---|
| Backend | Firebase Realtime Database (cloud) | **localStorage only** (this browser) — `demo_firebase_mock.js` replaces the Firebase SDK |
| Login | Email/password (Firebase Auth) | **Open access** — auto-signs-in a demo admin, no login screen |
| Data | Real figures entered by staff | **Invented sample figures** generated by `demo_seed.js` |
| Names | Real gangs / company / estate | Neutral demo names (Gang Alpha…Foxtrot, "Greenacre Plantations", "Demo Estate") |

Functionality is otherwise identical — every module, report, import/export and
calculation is the unmodified app code.

## How it works

1. `demo_firebase_mock.js` (loaded first) defines a `window.firebase` shim whose
   `auth()` / `database()` are backed by `localStorage` — nothing ever leaves the
   browser. On load it seeds the fake dataset (once) and auto-signs-in.
2. `demo_seed.js` builds the fake dataset. It reuses the app's **own** default-data
   generators (`getDefaultSprayingData`, `_manuringDefault2025`, `INITIAL_FFB_BUDGET`,
   `INITIAL_RAINFALL_2025`, the Iron Horse constants) as structural templates, then
   scrambles the numbers and swaps in demo gang names — so the shapes are always
   correct. Modules with no baked default (Planting, Harvesting Performance, Iron
   Horse expenses, Wages, Maintenance) are generated from scratch in the correct shape.
3. Weekly Activity is intentionally left empty (its photos/maps can't be faked).

Edits made in the demo persist in **this browser's** localStorage. Use the
**"Reset demo data"** button in the top ribbon (or run `window.__demoReset()` in the
console) to wipe edits and regenerate the original fictional dataset.

## Running it

Serve over HTTP (the Excel-template fetch needs a server), e.g. from this folder:

```
npx serve . -l 4178      # then open http://localhost:4178/
```

or host this folder on GitHub Pages / any static host.

## Known limitation

The downloadable Excel **reports** (YTD / Rainfall / Spraying) are built from the
templates in `Report samples/`. These have been scrubbed of corporate identity —
company name, registration number, estate name, document-author metadata, the
threaded-comment author registry, and leaked local file paths are all replaced with
neutral demo values.

One thing is intentionally **not** anonymised: a few worksheet **tab names** in
`Havesting Performance Dec 2025.xlsx` still carry real worker first-names (e.g.
`DARSO GANG`). Those tabs are wired into formulas, defined names and charts, so
renaming them safely needs Excel/LibreOffice to revalidate the links — out of scope
for the text-only scrub. Rename them in Excel if you need fully anonymised tabs too.
